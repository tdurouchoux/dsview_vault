GraphQL Federation is an architecture for building distributed GraphQL services. It allows multiple GraphQL services to be composed into a single unified schema, enabling clients to access data from multiple sources without needing to know how that data is partitioned. Federation works by defining entities and extending them across services, allowing for modular and scalable API design. This approach is particularly useful in microservices architectures where data is spread across multiple domains or services.
