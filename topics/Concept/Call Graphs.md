A call graph is a directed graph that represents the relationships between functions or methods in a codebase, showing which functions call other functions. It is a critical tool for understanding code structure, dependencies, and the potential impact of changes. In the context of software development and AI agents, call graphs help identify the 'blast radius' of a code change, enabling developers to assess risks, debug issues, and maintain system integrity. Call graphs can be built incrementally and updated in real-time as the codebase evolves.
