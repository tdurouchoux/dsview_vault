The History Database is a data structure used in Attention Residuals to store all intermediate outputs (states) generated by the model during forward propagation. Instead of passing a single tensor through the network, each layer appends its outputs to this database, allowing subsequent layers to query and utilize earlier computations. The database grows by two entries per layer (one from the multi-head attention block and one from the MLP block), enabling layers to access the full history of computations. This approach facilitates learned routing and improves gradient propagation.
