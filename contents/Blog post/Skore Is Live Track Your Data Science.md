## Summary

Skore, a new data science tool, is now publicly available. It is designed to address challenges in enterprise data science, such as technology-first thinking, spiraling costs, vendor lock-in, lack of industrial maturity, and fading scientific thinking. Skore is a collaboration layer for teams, built for and by data scientists, and is compatible with scikit-learn.

Key features of Skore include:

1. **Model Evaluation**: Evaluate any model in one line of code, with automatic generation of relevant metrics, feature importance, and plots.
2. **Cross-Validation**: Get a complete report for each fold of cross-validation, not just a score.
3. **Model Benchmarking**: Compare multiple models side by side in a structured way.
4. **Methodological Mistakes**: Catch potential modeling issues early by exploring variable associations and data splitting strategies.
5. **Project Organization**: Save reports, experiments, and artifacts in a structured way, locally or remotely.
6. **Team Collaboration**: Share, compare, and build upon each other's experiments, with visibility across a team's work.

Skore aims to reduce the time spent on documentation navigation, eliminate code boilerplate, and guide users toward the right methodological choices. It is designed for data scientists who use Python and the scikit-learn ecosystem, whether they work alone or in a team. The tool is open for feedback and contributions, with plans for future enhancements such as deeper guidance, AI-powered augmentation, full process coverage, and richer collaboration features.
