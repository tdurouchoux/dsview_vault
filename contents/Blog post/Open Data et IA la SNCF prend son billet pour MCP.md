## Summary

The SNCF is leveraging open data and AI to enhance its data valuation strategy. The company has three complementary approaches to data consumption: open data for the public, data sharing with trusted ecosystems, and an API for industrial uses. These tools are used both externally and internally, with notable examples including a notification system for TGV Max and energy consumption optimization in stations.

The SNCF is also exploring the use of the Model Context Protocol (MCP) to improve customer interactions. MCP, a protocol that standardizes communication between large language models (LLMs) and external tools, is being used to create a server that allows users to query SNCF services in natural language, including voice queries. The project is being developed in phases, with the first phase focusing on open data and the second phase integrating voice prompts. The launch of the MCP server has been slightly delayed but is still on track. The ultimate goal is to provide a comprehensive, personalized, and proactive travel information experience, transforming the customer relationship from transactional to conversational and intelligent. However, challenges remain, including data quality, contractual and security issues, and the risk of AI hallucinations.
