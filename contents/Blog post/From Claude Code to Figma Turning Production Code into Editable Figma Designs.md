## Summary

Figma now integrates with Claude Code to convert production UI code into editable Figma frames, bridging code-first and design-first workflows. Key technical points:

- **Capture-to-Figma**: Directly paste live UI (from localhost, staging, or production) into Figma as editable frames, preserving structure and states.
- **Collaboration**: Teams can annotate, compare variations, and explore flows without local builds or context switching.
- **Workflow benefits**:
  - Visualize entire systems (e.g., multi-step flows) for pattern/consistency checks.
  - Iterate rapidly (duplicate frames, rearrange steps) without rewriting code.
  - Shared context for designers, engineers, and PMs to align early.
- **Roundtrip integration**: Use the Figma MCP server to sync Figma designs back into code workflows (e.g., for AI-driven code generation).
- **Goal**: Reduce friction between prototyping (code) and exploration (design), enabling faster iteration and better product decisions.

*Context*: Part of Figma’s broader MCP (Model Context Protocol) ecosystem to unify AI-powered tools (e.g., Figma Make) with code workflows.
