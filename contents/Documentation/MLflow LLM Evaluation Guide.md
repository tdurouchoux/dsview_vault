## Summary

MLflow LLM Evaluation provides tools for evaluating large language models (LLMs) in various applications, focusing on metrics and methodologies distinct from traditional machine learning models. Key components include:

- **Model Evaluation**: Supports MLflow pyfunc models, registered models, or any callable Python model. Evaluation requires metrics and evaluation data, which can be in various formats (e.g., pandas DataFrame).

- **Evaluation Metrics**: Two main types are available:
  1. SaaS model metrics (e.g., OpenAI) for scoring.
  2. Function-based metrics (e.g., Rouge, BLEU) for traditional evaluations.

- **Default and Custom Metrics**: Users can select from predefined metrics based on model types (e.g., question-answering, text-summarization) or create custom metrics using the MLflow API.

- **Prompt Engineering**: A UI for prompt engineering is available, allowing users to refine how models interpret and respond to prompts.

- **Evaluation Process**: Users can evaluate models by logging them to MLflow or using custom functions. The evaluation results can be viewed programmatically or through the MLflow UI.

- **Static Datasets**: MLflow supports evaluating static datasets without re-running models, allowing for efficient assessments of model outputs.

- **Deployment Integration**: MLflow can evaluate models deployed on various platforms, including local AI gateways and Databricks.

Overall, MLflow LLM Evaluation streamlines the process of assessing LLM performance, providing flexibility in metrics and evaluation methods tailored to the unique characteristics of language models.
