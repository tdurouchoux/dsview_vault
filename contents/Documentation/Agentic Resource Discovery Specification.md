## Summary

ARD (Agentic Resource Discovery) is an open discovery protocol enabling AI clients to query available resources for tasks. It operates before invocation, helping clients find the right resource (e.g., agents, APIs, MCP servers) via discovery services, which are not centralized but can be run by enterprises or communities.

Key points:
- **Purpose**: ARD is a discovery layer, not an execution runtime (e.g., not MCP, A2A, or Skills).
- **Scope**: Resources include AI Catalog entries like agents, plugins, or workflows.
- **Governance**: Developed by a working group (Microsoft, Google, Hugging Face, etc.) under Apache 2.0 license.
- **Interoperability**: Designed to work with existing systems (e.g., chatbots like Claude, ChatGPT) and supports publishing/publishing guides.
