## Summary

Claude 4 prompt engineering best practices focus on optimizing interactions with Claude 4 models (Opus 4 and Sonnet 4) to achieve better results. Key techniques include:

1. **Clear Instructions**: Provide explicit and detailed instructions to guide the model effectively. Example modifications can significantly enhance output quality.

2. **Contextual Clarity**: Explain the rationale behind instructions to improve understanding and response accuracy.

3. **Vigilance with Examples**: Use aligned examples to promote desired behaviors and avoid undesired responses.

4. **Response Formatting Control**: Use positive phrasing to specify output format (e.g., tell what to do rather than what not to do) and XML formatting for structured outputs.

5. **Leveraging Thinking Capabilities**: Utilize Claude's ability to think through complex problems. Encourage reflection and iterative reasoning to improve results.

6. **Parallel Tool Execution**: Claude 4 excels at executing multiple operations simultaneously; provide prompts that explicitly encourage this.

7. **Temporary File Management**: Instruct Claude to clean up any temporary files created during execution to minimize clutter.

8. **Frontend Code Generation**: Encourage rich and detailed designs by specifying desired features and interactions succinctly.

9. **Migration Tips**: When transitioning from Sonnet 3.7 to Claude 4, be specific about output expectations and explicitly request features.

Overall, clarity and detail in prompting are crucial for maximizing the potential of Claude 4 models.
