## Summary

Lilypad documentation outlines the steps for building with Large Language Models (LLMs) using a structured workflow:

1. **Prototype**: Start by creating a functional LLM call using the Mirascope library.
2. **Vibe Check**: Assess the quality of the output; if unsatisfactory, adjust the prompt accordingly. Document prompts to ensure reproducibility.
3. **Annotate**: Systematically label data, including inputs, outputs, and relevant metadata for thorough analysis.
4. **Analyze**: Identify trends and common issues by comparing outputs across different versions to determine effectiveness.
5. **Optimize**: Implement changes based on analysis to resolve common failure points and improve LLM responses.
6. **Iterate**: Continuously repeat the analysis and optimization processes until the desired performance is achieved.

Lilypad integrates observability features, enabling data management, annotation, tracking, and versioning in one tool, aimed at enhancing LLM development workflows.
