## Summary

pyobsplot is a Python library that enables the creation of charts using Observable Plot within Jupyter notebooks, VSCode notebooks, Google Colab, and Quarto documents. It aims to provide a syntax similar to JavaScript for ease of use.

Key features include:
- Compatibility with Pandas and Polars DataFrame and Series objects.
- Ability to generate plots as Jupyter widgets or export them as SVG, HTML, PNG, or PDF.
- Offline functionality without dependencies on Observable runtime.
- Caching of data objects for efficiency.
- Automatic conversion of Python date and datetime objects to JavaScript Date objects.

Limitations include:
- Interactive features like tooltips are only available in widget format.
- Limited IDE integration for documentation and autocompletion.

Installation instructions and usage guidelines are provided, along with a quick start option in Google Colab. The library is built on contributions from Observable Plot developers and utilizes the anywidget framework for widgets.
## Links

1. [Observable Plot](https://observablehq.com/@mbostock) - A library for creating interactive visualizations in JavaScript, which is the foundation for pyobsplot.
2. [Quarto](https://quarto.org) - A scientific and technical publishing system that integrates with Jupyter notebooks and is used in conjunction with pyobsplot.
3. [Polars](https://pola.rs) - A fast DataFrame library for Python, which is used in the examples provided in the pyobsplot documentation.
4. [Apache Arrow](https://arrow.apache.org) - A cross-language development platform for in-memory data, which enhances the performance of data serialization in pyobsplot.
5. [Jupyter](https://jupyter.org) - An open-source web application that allows you to create and share documents that contain live code, equations, visualizations, and narrative text, compatible with pyobsplot.
